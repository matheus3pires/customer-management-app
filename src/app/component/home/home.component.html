Obrigado pelo seu interesse em fazer parte da maior empresa de tecnologia do Brasil. Abaixo será apresentado todas as informações necessárias para iniciar o seu teste.

Objetivo

  • Nome.
  • CPF.
  • Telefones (um ou mais).
  • Endereço (um ou mais).

Requisitos Funcionais

  • O cliente pode ter um ou mais telefones associados.
  • O nome do cliente deve ter no mínimo 10 caracteres.
  • O CPF deve ser válido e único no sistema.
  • Cada telefone deve estar em formato válido e exclusivo.
  • Não deve permitir dados duplicados (nome, CPF ou telefone).

Detalhamento sobre o BACKEND

  • Utilizar linguagem Java (versão mínima 17).
  • Utilizar banco de dados PostgreSQL.
  • Prover a estrutura do projeto via containers, usando docker.
  • Todo o código deve ser coberto por testes unitários.
  • Usar o modelo arquitetural REST para implementação das APIs.

Desejável

  • Swagger para documentação

Entrega

  • Suba o projeto em um repositório público no GitHub.
  • Inclua instruções de build e execução (README.md).
  • O projeto deve ser funcional ao rodar via docker-compose up.
